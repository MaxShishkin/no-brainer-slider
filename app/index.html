No Brainer Slider

A really simple css3 drop-in slider

Download or Contribute

This is a small css-driven slider with really simplistic javascript logic. The goal was to see how far i can get using css transitions with minimum number of classes and small-possible javascript acting as a state changer. You can use it as a simple drop-in solution while prototyping, as a blueprint for more complex work on top or use it standalone as is.

How to use

No Brainer Slider (Nbs) depends on jQuery (it's a jQuery plugin) and Modernizr for feature detection. Particulary, Modernizr is used to detect csstransitions, csstransforms and csstransforms3d features. Using Nbs is simple, here's an example.

HTML:


<div class="slider nbs-transition-flip-x">
    <div>Slide 1</div>
    <div>Slide 2</div>
    <div>Slide 3</div>
    <!-- etc. -->
</div>
<div class="nav-for-slider">
    <span class="slide-nav">1</span>
    <span class="slide-nav">2</span>
    <span class="slide-nav">3</span>
</div>

JavaScript:


$('.slider').noBrainerSlider({
    nav: $('.nav-for-slider'),
    autoplay: false
});

// use api to control the slider if needed
var nbs = $('.slider').data('noBrainerSlider');

nbs.gotoSlide(3);
nbs.setAutoplay(true);

Here we have a <div class="slider nbs-transition-flip-x">, every direct child of which is considered a slide. The important part here is the class name nbs-transition-flip-x that defines the type of animation to use. The slider class name is used just to target the slider from the script and can be whatever you want.

Below slider comes the navigation <div class="nav-for-slider">. Each child of the navigation container with the class name of slide-nav is considered a navigation item. Navigation items correspond to a particular slide by the order in which they come. E.i. the first found element with the class of slide-nav is used to activate the first slide. Navigation is optional.

Then we need to activate the slider. We do it by calling the noBrainerSlider() method on the slider. Several options can be passed to the method:

  • nav — jQuery object representing the navigation for the slider,
  • autoplay — change slides automatically, true/false value, defaults to false,
  • autoplayDelay — autoplay delay in ms.

You can control the slider by hand in your code, just grab the slider controller, that is stored as data-attribute on the slider element. Api contains just two functions: gotoSlide(number) to switch between slides (zero-based) and setAutoplay(bool) to turn autoplay on/off.

Class names avaliable for different transition effects (see examples above and no-brainer-slider.sass file):

  • nbs-transition-slide-x
  • nbs-transition-slide-x-on-top
  • nbs-transition-slide-x-off-top
  • nbs-transition-slide-y
  • nbs-transition-slide-y-on-top
  • nbs-transition-slide-y-off-top
  • nbs-transition-fade
  • nbs-transition-flip-y
  • nbs-transition-flip-x
  • nbs-transition-cube-x
  • nbs-transition-cube-y
  • nbs-transition-cube-z

Known issues

IE10 seem not to support transformStyle: preserve-3d and backface-visibility: hidden — some examples are broken.
